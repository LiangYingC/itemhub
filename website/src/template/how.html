如何開始使用 Itemhub
Step 1 - 新增裝置
至「裝置」頁新增裝置，輸入裝置名稱，選擇裝置類型後選擇要使用的 PIN 及 PIN 對應的功能，如: D3 為開關 D7 為感應器。
PS:目前僅提供3種類型裝置，後續會再新增其他裝置類型。
Step 2 - 下載程式碼並燒錄至裝置
2-1 下載程式碼
至裝置列表頁或明細頁可下載程式碼，系統會自動產生裝置對應的程式碼。
2-2 將程式碼燒錄至裝置
完成下載程式碼後，開啟 ino 檔案，其中 {SSID} 請改成你所使用的 wifi 帳號， {WIFI_PASSWORD} 請改成 wifi 密碼，完成後存檔並將所有檔案燒錄至裝置內。
2-3 確認裝置的連線狀態
完成程式碼的燒錄後，在裝置列表/明細頁即可看到裝置的狀態會切換成上線，即代表成功連線。
Step 3 - 查看並設定觸發條件
3-1 - 查看並設定觸發條件
裝置成功連線後，可於裝置列表頁或明細頁切換裝置開關或查看感應器回傳的資料，即時的操控裝置與查看裝置資料。
3-2 - 設定自動化觸發條件
於觸發頁填入觸發的名稱、觸發的事件條件與操作目標設定
範例為：
1. 濕度大於等於 65 執行開啟除濕機
2. 濕度小於 65 執行關閉除濕機
如何用 Itemhub 串接 Google Smart Home
Step 1 - 建立 oAuthClient
1-1 進入 oAuthClient 列表
1-2 建立 oAuthClient
建立新的 oAuthClient，留在這個畫面, 後面步驟需要畫面中的 Client Id 和 Client Secret
Step 2 - Google Actions Console
2-1 進入 Google Actions Console
2-2 點擊 New project 建立新的專案
  • 輸入專案名稱
  • 選擇語言和地區
2-3 建立 Smart Home Project
  • 選擇 Smart Home
  • 點擊 Start Build
2-4 選擇 Name your Smart Home action 開始設定
2-5 Invocation
選擇語言和 Google Smart Home 的顯示名稱，輸入完直接儲存。
2-6 Actions
Fulfillment URL 填入 https://itemhub.io/api/v1/google-smart-home 讓 Google Nest 知道要把相對應的要求丟到 Itemhub 哪一個網址。
2-7 Account Linking: 輸入 Itemhub 產生的 oAuthClient 資料
  • Client ID 填入剛剛在 Itemhub 上產生的 Client ID
  • Client Secret 填入剛剛在 Itemhub 上產生的 Secret
  • Authorization URL 填入 https://itemhub.io/api/v1/oauth
  • Token URL 填入 https://itemhub.io/api/v1/oauth/token
2-8 找到 Project Id
Step 3 - Itemhub 補上 oAuth Redirect URL
輸入 https://oauth-redirect.googleusercontent.com/r/{projectId} 儲存
